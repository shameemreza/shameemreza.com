Next.js is a [JavaScript](/strict-vs-loose-equality-in-javascript/) framework built on top of React. It handles the parts of a React app that React itself leaves to you: routing, server-side rendering, static generation, code splitting, and bundling. If you've ever tried to set those up by hand in a plain React project, you already know why Next.js exists.

## What is Next.js?


Next.js is a framework that allows you to build server-rendered React applications. This means that when a user requests a page on your website, the server will render the page and send the HTML to the user's browser. This differs from a traditional client-rendered React application, where the [JavaScript](/truthy-and-falsy-values-in-javascript/) runs in the user's browser and renders the page.

## Why use Next.js?


The main reason most teams pick Next.js is performance and SEO. Server-rendered pages arrive with HTML already in place, so the browser paints content before the JavaScript finishes loading. Search engines don't have to execute your app to see what's on the page. You also get code splitting, image optimization, and file-based routing out of the box, which saves you from writing a lot of boilerplate config yourself.

## Creating your first Next.js app

To get started with Next.js, you'll need Node.js and npm installed on your computer. You can download [Node.js](https://nodejs.org/en/) from the official website, and npm comes bundled with it.

Once you have Node.js and npm installed, you can create a new Next.js application using the `"create-next-app"` command. Open your command line and run the following command:

```
npx create-next-app my-app
```


This will create a new directory called `"my-app"` with the basic structure of a Next.js application.

Next, navigate into the `"my-app"` directory and start the development server by running the following command:

```
cd my-app
npm run dev
```


This will start the development server and open your application in a web browser. You should see a `"Welcome to Next.js"` page.

Now you have a basic Next.js application running. You can start building your application by editing the files in the "pages" directory. Each file in the `"pages"` directory corresponds to a route in your application.

For example, if you create a file called `"about.js"` in the `"pages"` directory, it will be available at `"http://localhost:3000/about"`.

**Note:** The above example is just a quick start guide, but there are many other features and functionalities of Next.js that you can use, like dynamic routing, static site generation, serverless deployment and many more. You can find the official documentation of Next.js here [https://nextjs.org/docs/getting-started](https://nextjs.org/docs/getting-started).

## Is Next.js beginner friendly?


Next.js is considered beginner-friendly for developers who are familiar with [React.js](https://reactjs.org/) and have a good understanding of JavaScript and web development concepts. [Next.js](https://nextjs.org/) provides a simple and easy-to-use structure for building server-rendered React applications, and its built-in features such as automatic code splitting and server-side rendering can help improve the performance of your application.

Next.js has a large community, so tutorials, Stack Overflow answers, and examples are easy to find. The `create-next-app` tool gets you to a working project in a single command, which helps when you just want to try something out without wiring up a build.

If you're brand new to React and web development, Next.js will feel like a lot at once. It assumes you know the basics of React and JavaScript. Learn React first, then come to Next.js. The framework is much easier when you're not fighting React and Next.js at the same time.

## Can I learn Next.js without knowing React?


You can technically learn Next.js without knowing React, but you'd be learning both at the same time and blaming Next.js for things that are actually React concepts. Components, state, props, and JSX all come from React. Next.js just adds file-based routing, server rendering, and a build pipeline on top.

Learn React first. Once you're comfortable with components and hooks, Next.js will feel like a natural extension rather than a separate framework to memorize.

## Is Next.js harder than React?


Next.js isn't harder than React so much as it's React plus extra things you eventually need anyway: routing, server rendering, code splitting, and a build pipeline. If you already know React, picking up Next.js is mostly about learning the conventions the framework chose for those concerns, not new JavaScript.

In practice, most of what makes Next.js feel harder is actually React being harder once it's deployed to real users. File-based routing and server rendering don't introduce new ideas so much as they force you to think about them.

## Where to go next

If you want more than a hello-world, two things will give you the most value early on: read the [official Next.js docs](https://nextjs.org/docs) end-to-end (they're short), and build one small project with data fetching from a real API. After that, everything else in the framework starts making sense in context.
